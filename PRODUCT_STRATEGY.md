# Nuucard Product Strategy & Roadmap

## 1. PoC (Proof of Concept) 검증 분석

### 핵심 가설 (Core Hypothesis)

> "사람들은 카카오톡보다는 무겁고, 손편지보다는 가벼운, **'중간 온도의 소통
> 방식'**을 원한다."

### Value Proposition Canvas

- **Customer Pain (고통)**:
  - 디지털 피로감 (Digital Fatigue).
  - 메신저의 가벼움으로 인한 진심 전달의 어려움.
  - 실물 엽서를 사서 보내는 과정의 번거로움.
- **Gain Creator (이득)**:
  - **고감도 비주얼**: 엄선된 사진으로 감성을 대변.
  - **프라이버시**: 클라이언트 암호화로 "둘만의 비밀 편지"라는 경험 제공.
  - **소유욕 충족**: 디지털로 받고 실물로 소장하는 하이브리드 경험.
- **Product Market Fit**: 감성을 해치지 않는 기술(암호화, 웹)을 통해 아날로그의
  불편함만 제거한 서비스.

### AARRR 퍼널 전략 (MVP 단계)

1. **Acquisition (유입)**:
   - 인스타그램(@nuucard) 콘텐츠 마케팅.
   - **Viral Loop**: 편지를 받은 수신자가 잠재적 신규 유저가 됨.
2. **Activation (활동)**:
   - **핵심 행동**: "편지 쓰기" 완료 및 링크 생성.
   - **KPI**: 랜딩 페이지 방문 대비 편지 작성 완료율 (Conversion Rate).
3. **Retention (유지)**:
   - **Lead Magnet**: 배경화면 다운로드를 통해 브랜드 리콜 유도.
   - **재방문 유도**: "내 사진 입점하기"를 통해 크리에이터로 전환 유도.
4. **Revenue (매출)**:
   - 현재 모델: 스마트스토어 링크 클릭 (Outlink).
   - **KPI**: `View Page` -> `Shop Link` 클릭률 (CTR).
5. **Referral (추천)**:
   - 서비스 구조 자체가 추천 메커니즘 (편지 발송 = 추천).
   - **전략**: 편지를 다 읽은 후 "나도 답장하기" 버튼의 가시성 확보.

## 2. 전략적 분석 (SWOT)

| Opportunities (기회)                                                                                                                                                     | Threats (위협)                                                                                                  |
| :----------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------- |
| - **연말/새해 시즌**: 12월~1월은 연하장 수요가 폭발하는 시기.<br>- **레트로 트렌드**: Gen Z의 아날로그/필름 감성 선호.<br>- **인스타그래머블**: 공유하고 싶은 UI 디자인. | - **낮은 진입 장벽**: 기술적으로 모방하기 쉬운 서비스.<br>- **일회성 사용**: 한 번 쓰고 잊혀질 가능성 (휘발성). |

## 3. 제안: 다음 단계 로드맵 (Next Steps)

### Phase 1: 데이터 기반 최적화 (즉시 실행)

- **목표**: Viral Loop의 효율 극대화 (수신자 -> 발신자 전환).
- **액션**:
  - GA4/Clarity 데이터를 분석하여 이탈 구간(Drop-off) 식별.
  - 편지 수신 페이지(`ViewPage`)에서 "나도 쓰러 가기" 버튼의 효율 테스트.

### Phase 2: 기능 확장 (단기 - 1~2개월 내)

- **Feature 1: "답장하기" 기능 강화**
  - 현재는 처음부터 다시 쓰는 구조. 받은 편지의 맥락을 이어가는 "Reply" 모드
    개발.
- **Feature 2: 예약 발송 (Time Capsule)**
  - "크리스마스 이브에 도착하도록 설정."
  - 기다림이라는 아날로그적 요소를 디지털 기능으로 구현.
- **Feature 3: 청각적 경험 추가 (Re-visit)**
  - MVP에서 제외했던 BGM이나 타자기 소리 효과음(ASMR)을 선택적으로 도입하여
    몰입감 증대.

### Phase 3: 비즈니스 모델 검증 (중기)

- **Direct Integration**: 스마트스토어 아웃링크 대신, 앱 내에서 배송지 입력 및
  결제 연동 (PG사 연동).
- **Premium Assets**: 유료 엽서 테마 또는 유명 작가 콜라보레이션 엽서 판매.
